## 修士論文 LaTeX テンプレート

修士論文用の LaTeX テンプレートです.

### ビルド方法（LuaLaTeX）

このテンプレートは `latexmkrc` を同梱しているので, 通常は以下でOKです.

```bash
latexmk main.tex
```

PDF生成後, 掃除する場合は:

```bash
latexmk -c
```

### 変更する場所

- `main.tex`
  - タイトル, 年度, 所属, 学籍番号, 氏名, 指導教員など（プレースホルダを置換）
  - 概要（`abstract.tex` を編集）
- 各章: `chapter1.tex` 〜 `chapter5.tex`
- 参考文献: `main.bib`
- 図: `fig/` 配下に配置（章ファイル内で `\includegraphics{...}` を使う）

### 一般ルール

#### ビルド
- `latexmk main.tex`（LuaLaTeXでPDFを直接生成）
- 生成物（例: `main.pdf`）はGit管理しない（`.gitignore` 対応済み）

#### 図（ファイルサイズと品質）
- 線画, 模式図: 可能ならベクター（PDF, SVG→PDF等）で出力する
- ヒートマップ等: PDFに埋め込むと重くなりやすいので, 必要に応じてラスタライズ（PNG等）してサイズを抑える
- PDFが大きい場合（例: 10MB級）は, どの図が肥大化要因か切り分けて最適化する
- 計算結果はHDF5やNumPy形式で保存する.
- 計算コードとプロットスクリプトを分ける.
- 再計算せずに図を調整できるようにする.
- (a), (b)などのラベルは, matplotlibやMakieのプロットスクリプトで直接描画する.
- KeynoteやPowerPointで後から貼り付ける方式は, 図の再編集時に手間が増える可能性がある.
- ベクター図の編集にはAffinity（無料で利用可能）を使う選択肢がある.

#### 本文（文章と構成）
- Gitで差分を見やすくするため, 原則として1文につき1行で書く. 複数の文を1行に詰め込まない.
- 用語は統一する（例: 「多電子」より「多体電子」など, 文脈に合う語を選ぶ）
- 曖昧な表現（例: 「〜ような」）を多用しない
- 段落間の論理のつながりを明示する

#### 表記（英字, 数式, 句読点）
- 句読点は `,` と `.` に統一する（Japanese comma/period variants are not allowed）
- 英字略語の直後に括弧で説明を付けるときは, 括弧の前に半角スペースを入れる  
  例: `Tensor Cross Interpolation (TCI)`
- 文末が数式で終わる場合は, 数式の後に `.` を打つ  
  例: \(E=mc^2\).

#### 結果の見せ方
- 2次元量などは, ヒートマップだけでなく代表的な断面図（固定したパラメータでのプロット）も用意する
